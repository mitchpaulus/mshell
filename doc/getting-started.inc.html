Getting started with mshell is a quick setup: install the CLI binary and then enable editor support.

Installation § Back to top

1) Download the binary and add it to your PATH

mshell ships as a single executable. Download the appropriate release archive from the GitHub releases page, extract it, and place the binary on your PATH.

Linux/macOS

tar -xvf linux_amd64.tar.gz
mv msh ~/.local/bin/
chmod +x ~/.local/bin/msh

Windows

# cmd.exe
tar -xvf windows_amd64.tar.gz
move msh.exe "%USERPROFILE%\\bin\\"

# PowerShell
tar -xvf windows_amd64.tar.gz
Move-Item msh.exe "$env:USERPROFILE\\bin\\"

2) Set up the standard library

Copy lib/std.msh from this repo into the startup location for the version of msh you are installing. msh v0.13.0 looks for the standard library at $XDG_DATA_HOME/msh/lib/v0.13.0/std.msh on Linux/macOS (falling back to ~/.local/share/msh/lib/v0.13.0/std.msh) and at %LOCALAPPDATA%\msh\lib\v0.13.0\std.msh on Windows.

It also loads a user init file from $XDG_CONFIG_HOME/msh/init/init.msh on Linux/macOS (falling back to ~/.config/msh/init/init.msh) or %LOCALAPPDATA%\msh\init\init.msh on Windows. If a script starts with VER "v0.13.0", msh uses the matching standard library path above and loads the init file from init/v0.13.0/init.msh instead.

Examples:

# bash/zsh/sh
mkdir -p "$XDG_DATA_HOME/msh/lib/v0.13.0"
cp lib/std.msh "$XDG_DATA_HOME/msh/lib/v0.13.0/std.msh"
mkdir -p "$XDG_CONFIG_HOME/msh/init"
mkdir -p "$XDG_CONFIG_HOME/msh/init/v0.13.0"
touch "$XDG_CONFIG_HOME/msh/init/init.msh"
touch "$XDG_CONFIG_HOME/msh/init/v0.13.0/init.msh"

# fish
mkdir -p "$XDG_DATA_HOME/msh/lib/v0.13.0"
cp lib/std.msh "$XDG_DATA_HOME/msh/lib/v0.13.0/std.msh"
mkdir -p "$XDG_CONFIG_HOME/msh/init"
mkdir -p "$XDG_CONFIG_HOME/msh/init/v0.13.0"
touch "$XDG_CONFIG_HOME/msh/init/init.msh"
touch "$XDG_CONFIG_HOME/msh/init/v0.13.0/init.msh"

# PowerShell (current session)
$libDir = Join-Path $env:LOCALAPPDATA "msh\\lib\\v0.13.0"
$initDir = Join-Path $env:LOCALAPPDATA "msh\\init"
New-Item -ItemType Directory -Force -Path $libDir, $initDir, (Join-Path $initDir "v0.13.0")
Copy-Item lib\std.msh (Join-Path $libDir "std.msh")
New-Item -ItemType File -Force -Path (Join-Path $initDir "init.msh")
New-Item -ItemType File -Force -Path (Join-Path $initDir "v0.13.0\\init.msh")

# Windows (GUI)
# Create %LOCALAPPDATA%\msh\lib\v0.13.0\std.msh
# and both %LOCALAPPDATA%\msh\init\init.msh
# and %LOCALAPPDATA%\msh\init\v0.13.0\init.msh.

An example installer script is available at install.sh.

GitHub Actions § Back to top

To use mshell in GitHub Actions workflows, use the official action: 

steps:
  - uses: mitchpaulus/mshell@v0.12.0
  - run: msh script.msh

To pin a specific version: 

steps:
  - uses: mitchpaulus/mshell@v0.12.0
    with:
      version: v0.12.0
  - run: msh script.msh

The action downloads the pre-built binary for the runner platform and installs the matching startup files automatically.

Editor Support § Back to top

Syntax highlighting and editor integrations are available for common editors:

  • VS Code Extension: mshell extension
  • Sublime Text: msh.sublime-syntax
  • Notepad++: user-defined language files (light and dark)
  • Vim/Neovim: mshell-vim

Shell completions for using mshell within a different shell can be generated by the CLI: 

# bash
source <(msh completions bash)

# fish
msh completions fish | source

# elvish
msh completions elvish | eval

# nushell
msh completions nushell | save --force $"($nu.default-config-dir)/completions/msh.nu"
use $"($nu.default-config-dir)/completions/msh.nu" *

A language server is bundled with the CLI, providing builtin hover information, completion on @ variables, and scope-aware variable renaming.
