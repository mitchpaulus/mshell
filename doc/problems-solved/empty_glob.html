Empty glob behavior in filename expansion

In traditional POSIX shells, if a glob doesn't match, you might get an error. There is also configuration that affects the behavior. The following parameters all affect the behavior of globs:

  • nocaseglob
  • globskipdots
  • nullglob
  • failglob
  • dotglob
  • $GLOBIGNORE

Do you know what all of these do? I certainly don't.

mshell is much simpler. glob is a function that takes a string and returns a list of strings. If there are no matches, you always still get a list, it is just empty.
