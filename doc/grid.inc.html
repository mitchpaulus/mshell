msh provides a high-performance columnar data frame type called Grid, inspired by pandas and Axon grids. Grids are ideal for working with tabular data, time series analysis, and data transformation pipelines.

Overview § Back to top

A Grid stores data in columnar format, where each column is stored as a contiguous array. This provides excellent cache locality for column-wise operations like aggregations, filtering, and transformations.

Grids support:

  • Typed column storage: Columns with homogeneous types (all integers, all floats, all strings, or all datetimes) use specialized storage for performance.
  • Grid-level metadata: Optional dictionary attached to the entire grid.
  • Column-level metadata: Optional dictionary attached to each column.
  • Lazy row access: The GridRow type provides efficient row-by-row access without copying data.
  • Views: The GridView type represents a filtered subset of rows without copying data.

For ingestion from delimited text or other row-oriented sources, use toGrid to convert [[str]] into a Grid. The first row is treated as the header row.

Literal Syntax § Back to top

Grids are created using the [| and |] delimiters.

Basic Grid

[|
   name, age, city;
   "Alice", 30, "NYC";
   "Bob", 25, "LA";
   "Carol", 35, "Chicago"
|]

Structure:

  • Column definitions come first (comma-separated names, terminated by ;)
  • Row data follows (comma-separated values, each row terminated by ;)
  • The trailing semicolon on the last row is optional

Empty Grid

[| col1, col2; |]  # Grid with columns but no rows

Metadata § Back to top

Grid Metadata

Optional grid-level metadata is specified as a dictionary followed by ; before the column definitions: 

[|
   { "source": "sensors", "version": 2 };
   timestamp, value;
   2024-01-01T10:00, 23.5;
   2024-01-01T11:00, 24.1
|]

Column Metadata

Each column can have optional metadata specified as a dictionary immediately after the column name: 

[|
   temperature { "unit": "celsius" }, humidity { "unit": "percent" };
   23.5, 65.0;
   24.1, 62.5
|]

Grid Operations § Back to top

Name Description Signature
gridRows Get the number of rows in a Grid or GridView. (Grid|GridView -- int)
gridCols Get the list of column names from a Grid or GridView. (Grid|GridView -- [str])
gridMeta Get the grid-level metadata dictionary. Returns none if no metadata. (Grid|GridView -- Maybe[dict])
gridColMeta Get the metadata dictionary for a specific column. Returns none if no metadata. (Grid|GridView str:colname -- Maybe[dict])
gridCol Extract a single column as a list of values. (Grid|GridView str:colname -- [a])
gridSetCell Set a single cell value. Returns the modified grid. (Grid str:colname int:rowIdx value -- Grid)
gridAddCol Add a new column to a grid. The third argument can be a list of values (must match row count) or a default value for all rows. (Grid str:colname [values]|default -- Grid)
gridRemoveCol Remove a column from a grid by name. (Grid str:colname -- Grid)
gridRenameCol Rename a column in a grid. (Grid str:oldname str:newname -- Grid)
select Project a Grid or GridView to the requested columns, preserving the requested order and returning a materialized Grid. (Grid|GridView [str] -- Grid)
exclude Drop the requested columns from a Grid or GridView and return a materialized Grid. (Grid|GridView [str] -- Grid)
derive Append a derived column to a Grid or GridView. The metadata dictionary is attached to the new column. (Grid|GridView str:colname dict:meta (GridRow -- a) -- Grid)
groupBy Group rows by key columns and return a summarized Grid. Aggregation specs are dictionaries with required "agg" quotation plus optional "name" and "meta". (Grid|GridView [str]:keys [dict]:aggs -- Grid)
updateCol Mutate a column in a Grid by applying a quotation to each cell. When used on a GridView, a new Grid is materialized from the viewed rows, the quotation is applied to that column, all result columns are retyped, and the backing Grid is left unchanged. The quotation must return exactly one non-container value for each cell. (Grid|GridView str:colname (a -- b) -- Grid)
gridCompact Materialize a GridView into a real Grid (copies the filtered rows). If already a Grid, returns it unchanged. (Grid|GridView -- Grid)
len Returns the number of rows in a Grid, GridView, or columns in a GridRow. (Grid|GridView|GridRow -- int)
join Inner equi-join of two Grids or GridViews using key extractor quotations on each side. Polymorphic with the string-join built-in: when the top of the stack is a quotation, the grid form is used. See Joins. (Grid|GridView Grid|GridView (GridRow -- a) (GridRow -- a) -- Grid)
leftJoin Left outer equi-join. Unmatched left rows appear with right-side cells filled with none. (Grid|GridView Grid|GridView (GridRow -- a) (GridRow -- a) -- Grid)
outerJoin Full outer equi-join. Unmatched rows from either side appear with the absent side filled with none. (Grid|GridView Grid|GridView (GridRow -- a) (GridRow -- a) -- Grid)

Examples

# Create a grid
[| x, y; 1, 2; 3, 4 |] grid!

# Or build one from parsed row data
"x,y\n1,2\n3,4\n" parseCsv toGrid csvGrid!

# Get row count
@grid gridRows  # 2

# Get column names
@grid gridCols  # ["x" "y"]

# Extract a column
@grid "x" gridCol  # [1 3]

# Add a new column
@grid "z" [10 20] gridAddCol  # Grid with x, y, z columns

# Set a cell value
@grid "x" 0 100 gridSetCell  # Sets grid[0]["x"] = 100

# Select columns in a specific order
@grid ["y" "x"] select  # Grid with columns y, x

# Exclude a column
@grid ["y"] exclude  # Grid with only x

# Derive a new column with metadata
@grid "sum" { "kind": "derived" } (:x? :y? +) derive  # Grid with x, y, sum

# Group rows and compute multiple aggregations in one pass
[| region, sales; "East", 10; "West", 5; "East", 7 |]
["region"]
[
   { "name": "n", "agg": (gridRows) }
   { "name": "sales_total", "meta": { "unit": "USD" }, "agg": ("sales" gridCol sum) }
]
groupBy  # Grid with region, n, sales_total

# Update a string column in place to typed values
@grid "x" (toFloat?) updateCol  # Mutates column x

GridView § Back to top

A GridView is a filtered view of a Grid that references the original data without copying. It maintains a list of row indices that match the filter criteria.

GridViews are created by the filter operation and can be used in most places where a Grid is expected. To materialize a GridView into a real Grid (with copied data), use gridCompact. 

# Filter creates a GridView
[| x; 1; 2; 3; 4 |] (:x 2 >) filter  # GridView with rows where x > 2

# Materialize to a Grid
gridCompact  # Now a real Grid with copied data

GridRow § Back to top

A GridRow is a lazy reference to a single row in a Grid. It does not copy data; instead, it stores a reference to the parent grid and a row index. This makes iteration over grids very efficient.

GridRows support dictionary-like access:

  • :colname getter syntax returns the value at that column
  • get operation works with string keys
  • toDict materializes the row to an actual dictionary
[| name, age; "Alice", 30 |] grid!

# Access first row
@grid :0:  # GridRow for first row

# Get column value from GridRow
:name  # "Alice"

# Or use get
@grid :0: "age" get?  # 30

# Convert to dict
@grid :0: toDict  # {"age": 30, "name": "Alice"}

Iteration Operations § Back to top

Name Description Signature
filter Filter rows using a predicate quotation. The quotation receives a GridRow and should return a boolean. Returns a GridView (no data copying). (Grid|GridView (GridRow -- bool) -- GridView)
map Transform rows using a quotation. The quotation receives a GridRow and should return a GridRow or dict. Returns a new Grid. (Grid|GridView (GridRow -- GridRow|dict) -- Grid)
each Iterate over rows for side effects. The quotation receives a GridRow. (Grid|GridView (GridRow -- ) -- )
toDict Convert a GridRow to a dictionary. (GridRow -- dict)

Examples

[|
   name, score;
   "Alice", 85;
   "Bob", 92;
   "Carol", 78
|] students!

# Filter: students with score >= 80
@students (:score 80 >=) filter
# GridView with Alice and Bob

# Map: add a "passed" column
@students (
   toDict
   dup "score" get? 70 >= "passed" set
) map

# Each: print each student's name
@students (:name wl) each
# Prints: Alice, Bob, Carol (one per line)

JSON Serialization § Back to top

Grids serialize to JSON as an array of row objects: 

[| x, y; 1, 2; 3, 4 |] toJson
# [{"x": 1, "y": 2}, {"x": 3, "y": 4}]

GridRows also serialize as JSON objects: 

[| x, y; 1, 2 |] :0: toJson
# {"x": 1, "y": 2}

Joins § Back to top

join, leftJoin, and outerJoin combine two grids by an equality key. The key is computed by a quotation on each side that takes a GridRow and returns a single value. This makes renamed keys, derived keys, and case-insensitive matches all expressible without renaming columns first.

Stack signature, top to bottom on entry: right key extractor, left key extractor, right grid, left grid. Both grid arguments accept a Grid or GridView.

Inner join

[| id, name; 1, "Alice"; 2, "Bob" |] l!
[| uid, score; 1, 90; 2, 80 |] r!
@l @r (:id?) (:uid?) join

Compound key

Return a list to use a tuple key. Use 2tuple to pack two stack values into a list. Both extractors must produce keys with the same shape. 

@trades @positions
   (dup :date? swap :region? 2tuple)
   (dup :dt? swap :rgn? 2tuple)
   join

Renamed key

@orders @customers (:customer_id?) (:id?) leftJoin

Outer join

Unmatched cells are none. Affected columns fall back to generic storage; this is intentional and documented. 

@l @r (:id?) (:uid?) outerJoin

Semantics

  • Equality. Type-aware, matching groupBy: an int 1 does not match a float 1.0 unless both extractors normalize.
  • Null keys. Quotation results that are none never match anything (SQL NULL ≠ NULL). Such rows are dropped from inner joins, and appear as orphans in left and outer joins.
  • Allowed key values. Any non-container scalar, or a flat list of scalars (treated as a tuple key). Nested lists, dicts, grids, or other container shapes raise an error.
  • Output column order. All left columns first in their original order, then all right columns. Column-level metadata is preserved per-column. The output grid carries the left grid's metadata.
  • Column collisions. If any column name appears in both grids the join errors before doing any work. Use select, exclude, or gridRenameCol first. Key columns are not auto-deduplicated because the extractors are opaque.
  • Multi-match. When duplicate keys exist on both sides, every left match is paired with every right match (Cartesian within the bucket).
  • Row order. Inner and left joins emit rows in left-grid order. Outer joins append right-grid orphans after, in right-grid order.

Cross product idiom

The extractor signature can express a Cartesian product by collapsing both sides to a single bucket: 

@left @right (drop 1) (drop 1) join

Combine with filter for arbitrary predicate joins. A dedicated crossJoin built-in may be added later as a clarity shortcut.
