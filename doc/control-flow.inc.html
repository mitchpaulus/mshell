mshell provides control flow constructs for conditional execution and looping.

if / else* / else / end § Back to top

The if block is the primary conditional construct. The condition is evaluated before if and popped from the stack. 

true if
    "condition was true" wl
end

Add an else branch for when the condition is false:

false if
    "true branch" wl
else
    "false branch" wl
end

Use else* and *if for else-if chains:

2 n!
@n 1 = if
    "one" wl
else* @n 2 = *if
    "two" wl
else* @n 3 = *if
    "three" wl
else
    "other" wl
end

Conditions can be booleans or integers. For integers, 0 is true (like Unix exit codes), non-zero is false. 

0 if "zero is true" wl end
1 if "not printed" wl end

iff § Back to top

iff is a postfix conditional that executes quotations based on a condition. It has two forms: with and without a false branch. 

# Two-argument form: condition (true-quote) iff
true ("was true" wl) iff

# Three-argument form: condition (true-quote) (false-quote) iff
false ("true" wl) ("false" wl) iff

iff is useful for inline conditionals:

@count 0 > (@items process) iff

loop § Back to top

loop repeatedly executes a quotation until break is called. 

0 i!
(
    @i 5 >= if break end
    @i wl
    @i 1 + i!
) loop

Output:

0
1
2
3
4

break § Back to top

break exits the innermost loop. 

0 i!
(
    @i 1 + i!
    @i 3 = if
        "breaking" wl
        break
    end
    @i wl
) loop
"done" wl

Output:

1
2
breaking
done

continue § Back to top

continue skips the rest of the current iteration and starts the next one. 

0 i!
(
    @i 5 >= if break end
    @i 1 + i!
    @i 3 = if continue end
    @i wl
) loop

Output (3 is skipped):

1
2
4
5

match § Back to top

match provides multi-way dispatch based on value, type, or structure. Arms are checked top-to-bottom until a match is found. 

"hello" match
    "hello" : "greeting",
    "bye"   : "farewell",
    _       : "unknown",
end wl # Output: greeting

Each arm has the form pattern : body , or pattern :> body ,. The separator determines whether the matched subject is consumed. The trailing comma on the last arm is optional. : consumes the matched subject before the body runs. :> preserves the matched subject on the stack when the body runs. This is independent of pattern kind and bindings.

If no arm matches and no wildcard _ arm is provided, a runtime error occurs.

Value Matching

Match against literal values — integers, strings, booleans, and paths:

42 match
    1  : "one" wl,
    42 : "forty-two" wl,
    _  : "other" wl,
end

Use :> when the arm body needs the matched subject, for example when matching on type before sending the value through another function:

[1 2 3] match
    list :> len str,
    str  :> len str,
    _    : "other",
end wl # Output: 3

Type Matching

Match against a type name. Supported type patterns: int, float, str, bool, list, dict, path, date, quotation, maybe, binary. 

42 match
    int : "integer",
    str : "string",
    _   : "other",
end wl # Output: integer

Wildcard

The _ pattern matches any value and acts as a catch-all:

"anything" match
    _ : "wildcard",
end wl # Output: wildcard

Maybe Destructuring

Match and destructure Maybe values with just and none patterns. The just pattern binds the inner value to a variable: 

"value" just match
    just v : @v,
    none   : "got none",
end wl # Output: value

Bindings and separator choice are independent, so preserving the subject is also valid:

{ 'key': 'value' } :key match
    just v :> ?,
    none   : "not found"
end wl # Output: value

Use just _ to match a Just value without binding:

"ignored" just match
    just _ : "just something",
    none   : "none",
end wl # Output: just something

List Destructuring

Match lists by shape and bind elements to variables:

# Exact match — binds each element
[1 2 3] match
    [a b c] : @b str,
    _       : "other",
end wl # Output: 2

# Empty list
[] match
    []  : "empty list",
    _   : "not empty",
end wl # Output: empty list

# Spread — head and rest
[1 2 3 4 5] match
    [head ...tail] : @head str,
    _              : "other",
end wl # Output: 1

# Wildcard in list — skip positions
[10 20 30] match
    [_ middle _] : @middle str,
    _            : "other",
end wl # Output: 20

...rest can also appear in the middle of the pattern. Items before it match from the front, items after it match from the back, and the spread binding receives everything in between.

[1 2 3 4 5] match
    [first ...middle last] : [@first @middle @last] (str) map " | " join,
    _                      : "no match",
end wl # Output: 1 | [2 3 4] | 5

Dict Destructuring

Match dictionaries by required keys and bind their values. Extra keys in the subject are allowed — only the specified keys must be present: 

# Matches — both keys present
{ 'name': "Alice", 'age': 30 } match
    { 'name': n, 'age': a } : @n,
    _                       : "missing fields",
end wl # Output: Alice

# Falls through — 'age' key missing
{ 'name': "Bob" } match
    { 'name': n, 'age': a } : @n,
    { 'name': n }           : @n,
    _                       : "other",
end
wl
# Output: Bob

Scoping

match uses the same variable scoping as if blocks — arm bodies have access to all outer variables and can modify them. Destructuring bindings (from just, list, and dict patterns) are added to the outer scope. Separator choice controls whether the matched subject is consumed. 

1 outerVariable!
10 match
    int :> @outerVariable + str,
    _   : "Not found",
end wl # Output: 11

Output: 11 (the subject 10 plus @outerVariable which is 1).

Prefix Quote Syntax § Back to top

Prefix quotes provide an alternative syntax for applying functions to quotations. The syntax is functionName. ... end (that is, the function name has a period '.' appended to the end), which is equivalent to (...) functionName. 

# Traditional postfix syntax
[1 2 3 4 5] (3 >) filter

# Prefix quote syntax
[1 2 3 4 5] filter. 3 > end

Prefix quotes work with any function that expects a quotation, including map, filter, each, and user-defined functions:

# Double each element
[1 2 3] map. 2 * end   # Result: [2 4 6]

# Print each item
["a" "b" "c"] each. wl end

Prefix quotes can be chained:

# Filter positives, then double them
[-1 2 -3 4] filter. 0 > end map. 2 * end   # Result: [4 8]

Prefix quotes can also be nested (one inside another):

# For each sublist, filter elements > 5
[[1 2 3] [4 5 6] [7 8 9]] map. filter. 5 > end end   # Result: [[] [6] [7 8 9]]

filter also works on dictionaries; the quotation receives each value and matching entries keep their original keys:

{ "a": 1, "b": 2, "c": 3 } filter. 1 > end str wl   # {"b": 2, "c": 3}

This is also useful to turn boolean operators and and or into a more traditional lookup infix format: 

true or. false end and. true end # Like (true | false) & true = true
